# 1C Configurator Dark Theme — Техническое резюме

> Дата: 2026-07-20  
> Контекст: Pet-project для создания темной темы и инструмента кастомизации конфигуратора 1С.  
> Цель: Инструмент, позволяющий пользователям самостоятельно выбирать и настраивать цветовые схемы конфигуратора без модификации файлов платформы.

---

## 1. Проблема

Конфигуратор 1С — нативное Win32-приложение (GDI/WinForms), работающее исключительно в светлой теме. Нативной темной темы не существует, что создает дискомфорт при длительной работе и плохо смотрится в контенте (стримы, туториалы).

**Почему это сложно:**
- Конфигуратор не использует CSS или системные темы Windows.
- Все цвета и отрисовка зашиты в код через GDI/GDI+.
- Многие контролы — кастомные (owner-draw), включая редактор модулей и отладчик.
- Обновления платформы регулярны и меняют внутреннюю структуру.

---

## 2. Рассмотренные подходы

| Подход | Суть | Плюсы | Минусы | Вердикт |
|--------|------|-------|--------|---------|
| **1C:EDT** | Использовать Eclipse-based IDE с нативной темной темой | Официально, легально, просто | Не конфигуратор; нет привычной отладки; не все инструменты | Альтернатива, не решение |
| **High Contrast Windows** | Системная тема Windows | Не требует кода | Ломает весь интерфейс ОС; конфигуратор подхватывает частично | Не подходит |
| **Патчинг бинарников** | Редактирование `.dll` / `.exe` платформы (IDA, x64dbg, Resource Hacker) | Теоретически полный контроль | Нарушает лицензию; ломается при каждом обновлении; риск сломать установку; невозможно дать пользователю выбор темы | **Отклонено** |
| **Инжектор + профили** | Отдельная утилита, которая внедряет DLL в процесс `1cv8.exe` и перехватывает отрисовку (`WM_PAINT`, `WM_CTLCOLOR`, `NM_CUSTOMDRAW`) | Не трогает файлы платформы; переключение тем в 1 клик; пользовательские JSON-профили; работает после обновлений 1С | Не все контролы покрасить; редактор модулей — отдельный челлендж | **Выбрано** |

## 3. Выбранная архитектура: 1C Configurator Theme Engine

```
┌────────────────────────────────────────────┐
│  1C Configurator Theme Engine (.exe)        │
│                                              │
│  ┌──────────────┐  ┌──────────────────────┐  │
│  │ GUI профилей │  │ Инжектор DLL         │  │
│  │ • Цвета      │  │ • Загрузка в 1cv8    │  │
│  │ • Шрифты     │  │ • Перехват WM_PAINT  │  │
│  │ • Скругления │  │ • Применение темы    │  │
│  │ • JSON импорт│  │   из профиля         │  │
│  └──────────────┘  └──────────────────────┘  │
│                                              │
│  ┌────────────────────────────────────────┐  │
│  │ Менеджер профилей:                     │  │
│  │  • Dracula.json                        │  │
│  │  • One Dark.json                       │  │
│  │  • Monokai.json                        │  │
│  │  • Custom (пользовательский)           │  │
│  └────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

### Почему инжектор, а не патчер

- **Обновления 1С:** патч слетает, инжектор продолжает работать.
- **Множество тем:** патч = один набор цветов, инжектор = сколько угодно JSON-профилей.
- **Пользовательская кастомизация:** GUI-редактор цветов → экспорт в JSON.
- **Безопасность:** не модифицирует файлы платформы, не триггерит антивирус/защиту.
- **Распространение:** один установщик с `.exe` + `.dll` + папка `profiles/`.

## 5. Технический стек

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| Инжектор | C# (.NET 8/9, WPF/WinUI) | GUI, управление профилями, запуск инжекции |
| Нативная DLL | C++ (WinAPI) | Низкоуровневый перехват оконных сообщений |
| Хуки | `SetWindowsHookEx`, `SetWindowSubclass` | Перехват `WM_PAINT`, `WM_CTLCOLOR`, `NM_CUSTOMDRAW` |
| Рендеринг | GDI+ (System.Drawing) / GDI (C++) | Замена цветов фона, текста, рамок |
| Конфигурация | JSON | Профили тем, импорт/экспорт, сообщество может делать свои |

## 6. План реализации (MVP → v1.0)

### Фаза 1. Инжектор (2–3 недели)
- Сканирование процессов `1cv8.exe`, определение конфигуратора.
- Внедрение DLL через `CreateRemoteThread` / `EasyHook`.
- Перехват `WM_ERASEBKGND` и `WM_PAINT` для главного окна и панелей.
- Покраска фона и текста в стандартных контролах (`Edit`, `ComboBox`, `ListBox`).

### Фаза 2. GUI профилей (1–2 недели)
- WPF-окно: список профилей, редактор цветов, color picker.
- Предпросмотр изменений в реальном времени.
- Импорт/экспорт JSON.

### Фаза 3. Расширенные контролы (2–3 недели)
- **Дерево метаданных** — перехват `NM_CUSTOMDRAW`, owner-draw нод.
- **Таблицы** (`SysListView32`) — кастомные цвета строк, заголовков.
- **Редактор модулей** — поиск Scintilla HWND, попытка применить цвета через `SCI_STYLESET*`.

### Фаза 4. Публикация (1 неделя)
- GitHub-репозиторий.
- Релиз: `.exe` + `ThemeHook.dll` + папка `profiles/`.
- README с инструкцией: «Скачай → Запусти → Выбери тему → Открой конфигуратор».

## 8. Альтернативы для быстрого результата (контент)

Если нужен быстрый визуал для стрима/скриншота без разработки:

1. **Windows High Contrast Black** — включается перед записью, выключается после. Конфигуратор частично подхватывает.
2. **MicaForEveryone / WindowBlinds** — сторонние тулзы для темизации рамок и фона Windows.
3. **OBS-фильтр** — цветокоррекция на уровне захвата окна. Костыль, но работает за 5 минут.

---

## 9. Ключевые выводы

1. **Патчинг бинарников — тупик.** Не масштабируется, ломается при обновлениях, нарушает лицензию, не дает пользователю выбор.
2. **Инжектор + JSON-профили — правильный путь.** Чистый WinAPI, не трогает файлы 1С, позволяет сообществу создавать свои темы.
3. **EDT — не замена.** Это другой инструмент. Для тех, кто хочет остаться в конфигураторе, нужен именно инжектор.
4. **Редактор модулей — главная неизвестная.** Остальное покрасить реально за 2–3 недели.
